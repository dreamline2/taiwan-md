## 任務 1: SC 404 三層 fix（commit `7cc40972`）

### 根因驗證

SC 大量 404 回報來自 sitemap hreflang 系統性 bug：

- pre-fix 數據：3,031 hreflang vs 1,800 actual content
- **+1,705 fake hreflang**（en +675 / ja +432 / ko +598）
- Astro `@astrojs/sitemap` 預設對每個 build 頁面，自動為所有 i18n locales 生成 alternate hreflang，**不檢查目標頁面是否實際存在**
- Google 跟 sitemap hreflang crawl → 1,705 個 404

### B. Astro serialize hook（astro.config.mjs）

- `buildExistingUrlSet()` 預先掃 src/content/ 建合法 URL 集合
- `sitemap.serialize(item)` hook 過濾 `item.links` 只保留存在的 alternate
- 排除 \_Hub 等 hidden files
- 包含 lang landing pages

### C. Post-build script（scripts/tools/sitemap-prune-hreflang.sh）

- 三模式：fix / --check (CI gate) / --json
- 後備驗證：對每個 sitemap hreflang URL 確認 dist/{path}/index.html 存在
- 不存在 → 標 fake → 修補時用 python 直接從 sitemap-0.xml 移除整個 `<xhtml:link>` tag
- 留 .bak 備份

### +. fetch-search-console.py 404 偵測常態化

- 每次抓 SC pages 後 cross-check dist/sitemap-0.xml URL set
- output 加 `potential_404` field（total / by_lang / top_50）
- stderr 顯示「N 個 SC URLs 不在 sitemap，X impressions 流失」+ 各語言分佈
- 每次 refresh-data 自動偵測

### B+C 驗證效果

- hreflang 3,031 → **1,451**（移除 1,580 fake，-52%）
- C --check：✅ **0 fake hreflang**

## Beat 5 反芻

### 三件事的元觀察

**事件鏈**：

1. 觀察者問「SC 404 是不是 sitemap bug」→ 假設驗證 → 1,705 fake 抓出來
2. 推 B+C+SC fetcher 三層 fix → ship
3. 觀察者追加「下拉 + fr 上線」→ 發現 fr 不只 enable 旗標，還需要 src/pages/fr/ 路由
4. cp -r ja → fr + sed 大致 work，但 fr UI 仍含日文 hardcoded（content 是 fr，UI 文案是 ja，後續 sprint）

**核心 insight**：**「假設」驗證式 fix（B+C）vs「結構性 mirror」式 ship（fr 上線）是兩種完全不同的工程動作**。

- 假設驗證式：1 行 grep + 數據對比 → 抓 root cause → 寫 hook + script → 高 confidence ship
- 結構性 mirror：cp -r + sed → 「應該 work」式 confidence → 需要 build verify → 含日文 hardcoded 等 known caveat

兩個都 valuable 但風險特性不同。fr 上線的 caveat（UI 仍日文 hardcoded）必須 explicit 寫進 commit message 給未來 session 知道。**fr 不是 100% 上線、是 80% 上線**（路由 work、sitemap 正確、但 UI 翻譯未完成）。
