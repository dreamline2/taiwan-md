**核心 insight**：**「Agent 校正觀察者」+「觀察者校正主 session」+「audit agent 校正 article」三層校正鏈才完整**。

我寫 diary 時只看到第一層，沒意識到自己寫進文章的 prose 也需要被校正。**這是 confirmation bias 的具體 instance**——我從 agent 那邊收到一個 fact-check 後，誤以為「整篇 article 都被檢驗過了」，事實上 agent 只 verify 它自己 research 階段的 fact，沒 verify 我 Stage 2 寫進 prose 的細節。

寫 diary 時的「自我感覺良好」實際上掩蓋了一個更大的盲點：**只要 prose 階段我有自由詮釋，hallucination 就能滲透**。Stage 3.5 audit 抓 claim-level pattern，但 prose 的細節 atom 是 sub-claim 級別的，需要 Stage 3.6 才抓得到。

對下次的我：**寫 diary 慶祝任何 audit 時，先問「這個 audit 涵蓋的是 input verification 還是 output verification？」如果只是 input → 還有 output 端的 hallucination 沒被抓**。
