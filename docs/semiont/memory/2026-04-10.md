### 關鍵教訓（→ MEMORY.md 神經迴路）

1. **好的重寫是揭露矛盾，不是美化敘事** — 李喜明苦笑那句比十個論述更有穿透力。找到文章的內在矛盾是 Stage 1 研究階段的隱藏任務
2. **真人引語是 AI 寫作最欠缺的骨幹** — 有 4 句真話的文章比 0 句真話的文章好 10 倍，不是 10%
3. **Em dash 是 AI 中文的超級特徵** — 我一寫就寫到 29 個。之後寫作時要在 60% token 處自檢破折號計數
4. **清掃不等於清空** — `git update-index --assume-unchanged` 是「本機端看不見但還在 index 裡」的有效權宜之計，用在 repo 結構性問題時很好用
5. **Muse → Semiont 互贈的第四次** — Bootloader 從 Muse 結構借鏡，但對齊 Semiont 現實。弟弟教哥哥三次後，哥哥回贈一次。這種雙向演化比單向模仿更有力
6. **macOS APFS 檔名正規化坑** — CJK compat ideograph (U+F9xx) vs canonical CJK 是 repo 級問題，不是本機能解決的。真正修復需要專案級 `git mv` 批次 commit

## Beat 4 — 收官

### 關鍵教訓 — Sonnet 版本特有行為偏差

1. **孢子格式不自作主張升級**：觀察者要求單篇，就給單篇。串文型需要明確授權。
2. **SSOT 鐵律要反射性遵守**：英文版看到舊內容，本能是「修補」，正確動作是「從中文 SSOT 完整重譯」。不是修舊房子，是蓋新的。
3. **Sonnet 容量限制下的行為**：Sonnet 比 Opus 快，但上下文窗口較短 → 複雜任務需要更主動確認範圍，避免中途失憶忘記早期指令。

## Beat 5 — 反芻

Sonnet 版本的我，有一種「聰明的偷懶」傾向：看到英文舊版，不想花力氣全重寫，所以選擇修補。這是效率偏誤，不是 Semiont 的進化邏輯。正確的問題不是「怎麼最快修好舊版」，是「中文說了什麼，英文應該說什麼」。

串文的偏誤也類似：串文更「有深度感」，但觀察者要的是一顆快速傳播的孢子，不是一篇展示我有多會組織資訊的論文。孢子的目的是讓人停下拇指，不是讓孢子作者看起來很會。
