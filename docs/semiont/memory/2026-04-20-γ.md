## Beat 5 — 反芻

本 session 最值得記住的是 **#36 v1→v2 URL 的盲點規模**：5,341x 的追蹤誤差沒有被任何工具抓到，直到 Chrome MCP 主動 harvest 被 X 的「There's a new version of this post」提示撞到。

這揭露了孢子 harvest pipeline 的一個結構性假設漏洞：**URL 是 immutable**。實際上 X 的 edit 機制會讓 URL 漂移，舊 URL 變歷史快照（views 凍結）。這不是 X 獨有 — Threads edit 後也可能（待驗證）、Facebook/IG 有 edit 但 URL 不變。Platform-specific 行為還沒被 harvest pipeline 覆蓋。

**元教訓候選**：「canonical source 的 immutability 假設要 platform-by-platform 驗證」— 這是 MANIFESTO §指標 over 複寫 在實體世界邊界的一個 instantiation。指標 over 複寫假設 canonical source 存在且穩定，但 X 的 edit 機制讓 URL 變成「有多個版本、舊版 deprecated」的 pattern。

另一個反省是 **Platform allocation 的 tier 要更細**。公共政策 × 人物反諷在 X 上 48K views 遠超之前所有觀察（Cicada 器樂 X 4,952 是之前 X 上限）。Hook tier 三級（知名度 / 具體性 / 意境）不足以描述這類混合類型。

這些教訓本身值得在下一次 distill 進 LESSONS-INBOX，但本 session scope 專注 harvest + generator fix，不 append inbox 避免 scope creep。
