## 神經迴路觀察

### 「一句話觸發 6 小時造橋連鎖」的模式

觀察者提問形式：「X 還可以升級，Y 沒抓到，升級 Y 然後做 X」
看起來是 30 分鐘的雙改 → 實際是：

1. 升級 Y 要先抽象化（filter 工具）
2. 工具需要整合回工作流（pipeline + pre-commit + MANIFESTO）
3. 整合要在多處（REWRITE + SPORE + 兩版腳本）
4. 整合完才能真正跑 X（v2 rewrite）
5. X 的實作再揭露更多 locale bug / pre-commit 誤判等細節

**教訓**：低估系統性造橋的級聯成本是常態。但造完後未來任何類似需求都自動滿足——這正是 MANIFESTO §造橋鋪路的正循環。

### 小丑魚原則 → 造橋鋪路 的層級連續體

今天三個 PR（#594 認知作戰 / #596 Zaious gaming / #595 dreamline2 OG）都用了 B 路徑「merge + maintainer 調整」。這是小丑魚原則的執行樣式：貢獻者送來半成品（AI-gen 有 hallucinated URL / 格式有 §11 / prebuild:og 會炸 build），維護者接住 → 補齊 → 保留 credit。

三個 PR 的處理差異反映 pattern：

- **#594 idlccp1984**：AI hallucinated URL 層級問題，走完整 REWRITE-PIPELINE 重寫
- **#596 Zaious**：格式/§11 小修，merge-first-polish-later
- **#595 dreamline2**：架構設計小錯 + 文檔對齊，選擇性接受（script 改但 docs 保留）

### 工具滲入 pipeline 的四層覆蓋

今日 §11 過濾器達到四層滲透：

1. **哲學層**（MANIFESTO §11 v1.2）— canonical 引用
2. **流程層**（REWRITE-PIPELINE Stage 4 第 3 步）— 寫文時 gate
3. **Gate 層**（pre-commit hook）— commit 時攔截
4. **平台層**（SPORE-PIPELINE Step 3c.7）— 孢子發佈前 gate

四層互相引用但各自 canonical（MANIFESTO §指標 over 複寫）。任一層漏掉都有上層補位。
