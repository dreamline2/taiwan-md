### 架構決策（選 C 並統一到 shot=1）

**雙管線 → 單管線**：spore 與 OG 共用 **article page `?shot=1` 模式**（poster-style hero、justfont rixingsong-semibold、hide chrome）。獨立 `/og/[...path].astro` 路由**保留不刪**但不再是 OG meta 主要來源。

**為什麼這樣選**：

1. 一處維護 — spore 視覺品質 = OG 視覺品質，品牌一致
2. 不會出現「spore 好看但 OG 樸素」或反向的美學漂移
3. article hero 本身就含 title + description + breadcrumb，OG 三要素齊全
4. 日星鑄字體的品牌資產直接複用

**為什麼 JPG 85 取代 PNG**：

- 1200×630 漸層背景 + 中文字 → JPG vs PNG 在 FB/Threads/X 視覺差別為零
- 體積 -70%（150-300 KB PNG → 40-80 KB JPG）
- `og:image:type` SEO.astro 早就是 `image/jpeg`（line 386），改成 JPG 等於類型對齊

**測試驗證**：李洋.jpg = **37 KB**（vs 估計 PNG ~200 KB），單張 3.1s（含 server warmup），後續每張 ~1.5s。

### 神經迴路觀察

**小丑魚原則應用層深化**：α session 處理 #587 時走「授權邊界」判準（politically sensitive → escalate），本 session α 初判的 #590（request changes）跟 #592（escalate 觀察者）被觀察者明確修正為「小丑魚原則優先」——意思是：

> 對善意貢獻者溫和 = 維護者自己承擔事實查核 + 架構優化工作，不把這個負擔推回貢獻者

這跟「merge-first-polish-later」是同一棵樹的兩條枝幹：

- **merge-first**: 不讓貢獻者等待完美
- **小丑魚**: 不讓貢獻者承擔本屬維護者的驗證負擔

本次 #590 的王福瑞生年 verify，意外挖出兩個更大的事實錯誤——這證明了小丑魚原則的隱性收穫：**自己做事實查核時會觸發「順手驗證同段落其他宣稱」的連鎖反應**，而 request-changes 會讓這個連鎖反應不發生（貢獻者只補自己那一小塊來源）。

**DNA 候選升級**：「escalation 前先問『這個問題維護者能自己解嗎』」— 若能，小丑魚；若不能（政治立場、架構決策），才 escalate。
