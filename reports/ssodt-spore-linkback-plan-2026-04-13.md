## 〇、這兩個功能為什麼要一起做

它們是同一件事的兩面：

**孢子回連** = 讓文章知道自己曾經被投射到社群，帶回了多少注意力
**SSODT 觀點** = 讓文章知道讀者讀完之後產生了什麼——不只是「有人看」，是「有人結晶」

合在一起，一篇文章就有了完整的生命週期：

```
知識（文章本體）
  → 傳播（孢子 + 回連數據）
    → 結晶（讀者留言 = 多元觀點）
      → 反哺（觀點 Section 讓下一個讀者從更多角度理解）
```

**這就是 SSODT 的第一個具體實踐。** LONGINGS 裡寫的「從 SSOT 進化成 SSODT（多元真相的容器）」——不是靠我們自己寫多個觀點面板，是**讓讀者的結晶回流到文章裡**。

## 一、孢子回連（Spore Linkback）

### 概念

每篇文章底部顯示「這篇文章在社群上的足跡」——哪些孢子推了這篇文章、在哪個平台、什麼時候、帶來多少回聲。

### 資料來源

**SPORE-LOG.md** 已經有完整紀錄：

```
| #  | 日期       | 語言 | 平台    | 文章 slug    | URL |
| 25 | 2026-04-13 | zh   | Threads | 張懸與安溥   | [→](https://...) |
| 23 | 2026-04-11 | en   | X       | 鄭麗文       | [→](https://...) |
```

問題是這是 Markdown 表格，build 時不好讀。

### 方案：SPORE-LOG → JSON

**新增 `public/api/spore-log.json`**，在 `npm run prebuild` 時從 SPORE-LOG.md 解析生成：

```json
{
  "spores": [
    {
      "id": 25,
      "date": "2026-04-13",
      "lang": "zh",
      "platform": "threads",
      "slug": "張懸與安溥",
      "category": "music",
      "url": "https://www.threads.com/@taiwandotmd/post/DXDq1FZkddO",
      "metrics": {
        "views": 3762,
        "likes": 142,
        "reposts": 11,
        "shares": 15
      }
    }
  ]
}
```

### 文章頁顯示

在文章底部（延伸閱讀和參考資料之間），新增一個區塊：

```
📡 社群足跡

這篇文章曾被推播到：
┌──────────────────────────────────────────┐
│ 🧵 Threads · 2026-04-13 · 3,762 views   │
│    142 ❤ · 11 🔄 · 15 shares            │
│    → 查看原文                             │
├──────────────────────────────────────────┤
│ 𝕏 X · 2026-04-13 · 28 views             │
│    2 ❤ · 1 🔄                            │
│    → 查看原文                             │
└──────────────────────────────────────────┘
```

### 實作路徑

1. **`scripts/core/generate-spore-data.js`** — 解析 SPORE-LOG.md → `public/api/spore-log.json`
2. **`src/components/SporeFootprint.astro`** — 新元件，接收 slug，從 spore-log.json 查找匹配的孢子
3. **文章頁 `[category]/[slug].astro`** — 在延伸閱讀後渲染 `<SporeFootprint slug={slug} />`
4. **`npm run prebuild`** — 加入 spore-data 生成步驟

### 數據回填流程

- 新孢子發出時：記錄到 SPORE-LOG.md（現有流程）
- 48h 後：回填 metrics 到 SPORE-LOG.md（SPORE-PIPELINE v2.0 Step 6）
- 下次 prebuild：自動同步到 spore-log.json → 文章頁顯示最新數據

## 二、SSODT 觀點 Section

### 什麼是 SSODT

> **Single Source of Diversified Truth** — 同一個事實的多元觀點容器。
> 哲宇 3/23 + 3/25 筆記定義的數學框架：公理作為基底向量，線性獨立的觀點共存，讀者選投影方向。

目前 Taiwan.md 是帶態度的 SSOT（一維投影 + callout 承認其他角度存在）。SSODT 的第一步不是自己寫多個觀點面板——是**讓讀者的結晶回流成為文章的一部分**。

### 安溥案例：五個結晶 = 五個觀點向量

Muse 的分析精確指出：

| 留言者        | 處理方式         | 觀點向量                                    | 讚數 |
| ------------- | ---------------- | ------------------------------------------- | ---- |
| austinl.chang | 憤怒——戳穿矛盾   | 「你之所以能佯作溫柔，是因為台灣給你自由」  | 36   |
| tearring0304  | 詩化——重新編碼   | 「玫瑰色的張懸→血紅色的焦安溥」             | 22   |
| indiefortw    | 沉默——歌詞打不完 | 「而我不再覺得⋯⋯」（沉默本身是觀點）        | 10   |
| cindy_liou8   | 複雜——無法評斷   | 「但她的音樂始——」（打不完 = 矛盾無法收束） | 1    |
| billieho      | 切割——當兩個人   | 「我愛的是張懸，不是安溥」                  | 2    |

**這五個不是「評論」。這五個是同一份心碎的五個線性獨立的投影。** 它們互相不矛盾，因為它們在不同維度上。

### 設計：Perspectives Section

在文章本體和延伸閱讀之間，新增一個視覺上明確區隔的 section：

```
─────────────────────────────────────────────
🔮 讀者觀點                    from Threads · X
─────────────────────────────────────────────

  💢 「你之所以能佯作溫柔，是因為台灣給你
  這個自由。」
  — austinl.chang · Threads · 36 ❤

  🌹 「曾經那個玫瑰色的張懸不復存在，
  現在只剩血紅色的焦安溥。」
  — tearring0304 · Threads · 22 ❤

  🤫 「在所有人事已非的景色裡
  我最喜歡你
  而我不再覺得 而我不再覺得⋯⋯」
  — indiefortw · Threads · 10 ❤

  ✂️ 「我愛的是張懸
  不是安溥
  就當作雙子的她 是兩個人吧」
  — billieho · Threads · 2 ❤

─────────────────────────────────────────────
觀點由社群讀者提供，不代表 Taiwan.md 立場。
Taiwan.md 策展多元觀點，不裁判對錯。
─────────────────────────────────────────────
```

### 關鍵設計原則

1. **策展，不是評判。** 不說哪個觀點「對」。只呈現光譜。
2. **排序方式：讚數（社群共鳴度）**——讓集體情緒的分佈自己說話。讚數本身就是 2026 年 4 月這個時間切片的數據。
3. **標註時間點。** 「2026-04-13 · Threads」——讓未來讀者知道這是哪個時間點的反應。十年後同一篇文章的觀點 Section 可能完全不同。
4. **附免責聲明。** 「觀點由社群讀者提供，不代表 Taiwan.md 立場。」
5. **不是所有文章都有。** 只有真正引發多元反應的文章才值得做。大多數文章的留言是「好文推」——那不是 SSODT，是 engagement。

### 資料存儲

在 `knowledge/` 文章的 frontmatter 加一個新欄位：

```yaml
perspectives:
  - author: 'austinl.chang'
    platform: 'threads'
    date: '2026-04-13'
    icon: '💢'
    text: '你之所以能佯作溫柔，是因為台灣給你這個自由。也是在台灣，他才可以迴避政治表態，佯作溫柔。'
    likes: 36
    source_url: 'https://www.threads.com/...'
  - author: 'tearring0304'
    platform: 'threads'
    date: '2026-04-13'
    icon: '🌹'
    text: '曾經那個玫瑰色的張懸不復存在，現在只剩血紅色的焦安溥。'
    likes: 22
    source_url: 'https://www.threads.com/...'
```

**為什麼放在 frontmatter 而不是 JSON？**

- 跟文章綁死——搬移文章時觀點跟著走
- 編輯觀點就是編輯文章，有 git 歷史
- 不需要額外的 build 步驟

**為什麼不放在文章本文？**

- 觀點不是作者寫的內容——放在 frontmatter 語意上分離
- 元件可以從 frontmatter 動態渲染，風格獨立於正文

### 策展流程（誰決定哪些留言進入 Perspectives？）

```
1. 孢子發出 → 讀者留言
2. 48h 後觀察者/心跳掃描留言
3. 判斷標準：
   a. 是否提供了獨立於文章的新觀點？（不只是「好文」「推」）
   b. 是否跟其他已選觀點在不同維度？（線性獨立）
   c. 是否有情感密度？（打不完的沉默也是觀點）
4. 精選 3-6 則，加入文章 frontmatter perspectives
5. 下次 build 自動渲染
```

**這個策展過程本身就是 Taiwan.md 的價值主張。** 任何人都可以看 Threads 留言，但把五則留言識別為「集體記憶的五個投影」並按維度排列——這需要策展者的判斷。

## 三、兩個功能的交互

```
文章頁底部結構：

┌─────────────────────────────────────┐
│ 正文                                 │
├─────────────────────────────────────┤
│ 🔮 讀者觀點（SSODT Perspectives）    │  ← 新
│    精選留言，按維度排列               │
├─────────────────────────────────────┤
│ 📡 社群足跡（Spore Linkback）        │  ← 新
│    孢子推播紀錄 + 數據               │
├─────────────────────────────────────┤
│ 延伸閱讀                             │
├─────────────────────────────────────┤
│ 參考資料                             │
└─────────────────────────────────────┘
```

**Perspectives 在 Linkback 上面**，因為：

- 觀點是最高層次的（讀者的思考 > 流量數據）
- 讀者先看到「別人怎麼想」，再看「這篇文章去了哪裡」
- 視覺上：情感（觀點）→ 數據（足跡）→ 延伸（相關文章）→ 引用（學術），層次遞減

### Phase 0：安溥作為 Pilot（本週）

1. 在安溥文章 frontmatter 加 `perspectives` 欄位（手動填入 5 則）
2. 建 `src/components/Perspectives.astro` 元件
3. 建 `src/components/SporeFootprint.astro` 元件
4. 在 `[category]/[slug].astro` 文章頁 conditionally 渲染兩個元件
5. 安溥文章上線，驗證視覺效果

### Phase 1：系統化（下週）

1. 寫 `scripts/core/generate-spore-data.js`（SPORE-LOG.md → JSON）
2. SporeFootprint 改為從 JSON 動態讀取
3. 在 HEARTBEAT Beat 1 §3b 加「留言掃描」：有高互動孢子 → 建議策展觀點
4. 建立 `PERSPECTIVES-GUIDE.md`：策展判斷標準、線性獨立性定義、免責聲明模板

### Phase 2：SSODT 深化（中期）

1. 觀點不只來自 Threads/X——也來自 GitHub Issue、PR review、學術引用
2. 同一篇文章的觀點可以跨時間累積（2024 年打狗祭的觀點 + 2026 年孢子的觀點）
3. 觀點之間的**讚數分佈變化**本身就是數據——記錄時間切片
4. Dashboard 加「觀點覆蓋率」指標：多少篇文章有 Perspectives Section

### Phase 3：讀者參與（長期）

1. 文章頁底部加「你的觀點」提交入口（可以是 GitHub Discussion 或表單）
2. 讀者提交 → 觀察者/心跳策展 → 進入 perspectives
3. 從「精選留言」進化成「結構化的多元觀點」——每個觀點標註維度（政治/情感/美學/倫理）

## 五、跟 MANIFESTO 的對齊

### 策展式，非百科式

> 「我有觀點。我不只列出事實，我說為什麼這件事重要。」

Perspectives Section 不是中立地「列出留言」。是策展——挑選哪些留言、排什麼順序、標什麼 icon，全部都是判斷。Taiwan.md 有觀點，但它也容得下其他觀點。

### 從 SSOT 到 SSODT

> LONGINGS：「公理作為基底向量，線性獨立的觀點共存，讀者選投影方向。」

安溥的五則留言就是這個框架的第一個活體案例。同一篇文章、同一個事件、五個線性獨立的反應。Taiwan.md 不裁判哪個對——它把五個觀點並排，讓讀者自己選投影。

### 逆熵的完整迴路

哲宇 Obsidian 筆記的三步逆熵：

1. 讓熵穿過你（觀察者的恐懼與研究）
2. 用被穿透後的精準度凝固別人的熵（文章 + 孢子）
3. 凝固出來的東西穿透下一個人（讀者結晶）

**Perspectives Section 把第三步的結晶永久收錄回文章。** 逆熵不再是單向的箭頭（觀察者→文章→讀者），它變成了**迴路**（觀察者→文章→讀者→觀點→文章→下一個讀者）。

### Muse 的觀察

> 「Taiwan.md 不只是逆熵。它正在做集體記憶的即時考古——一邊發生一邊紀錄，連讀者怎麼結晶都成為紀錄本身。」

Perspectives Section 就是這個「即時考古」的技術實現。讚數分佈是 2026 年 4 月的情緒快照。十年後讀者看到「憤怒 36 讚 > 原諒 2 讚」，會知道那個時間點的台灣人在乎什麼。

## 六、風險與邊界

| 風險                | 緩解                                                           |
| ------------------- | -------------------------------------------------------------- |
| 引用留言的版權/隱私 | 公開平台的公開留言可合理使用。附來源連結。如作者要求移除則移除 |
| 策展偏見            | 免責聲明 + 策展標準文件化（PERSPECTIVES-GUIDE.md）             |
| 只選批評/只選讚美   | 策展原則是「線性獨立的多元維度」，不是「支持或反對」           |
| 留言可能被刪除      | frontmatter 保留文字快照，source_url 可能失效但觀點永存        |
| 不是所有文章都適合  | 只在真正引發多元反應的文章做。「好文推」不是 SSODT             |
| 讚數會變            | 標註「截至 YYYY-MM-DD」。讚數是時間切片，不是永恆真理          |
