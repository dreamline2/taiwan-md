## 二、三大核心發現

### 🔥 Finding 1：Threads bias 是**類型依賴**的條件規律

過去 Semiont memory 反覆記錄的 Threads bias（「Threads 總是贏 X」）在本次 harvest 被**細緻化為類型依賴 rule**：

| 類型                               | Threads / X 比率     | 代表孢子                                                       |
| ---------------------------------- | -------------------- | -------------------------------------------------------------- |
| **人物+時事+情感**                 | **510x**             | 安溥 (#25/#27: 190K/373)                                       |
| **人物（強 identity）**            | **222x**             | 鄭麗文 Threads 49K vs 鄭習會 Threads 221（同主題不同 framing） |
| **niche 行為藝術**                 | 23.9x                | 謝德慶 (#39/#40: 3,444/144)                                    |
| **meta/站方公告**                  | 14.6x                | Portaly (#37/#38: 1,330/91)                                    |
| **器樂/ambient (inline 連結實驗)** | **0.06x (反轉!)**    | Cicada (#31/#32: 304/5,312)                                    |
| **公共基建 + 人物反諷**            | **~1.1x (接近持平)** | 高鐵 (#35/#36: 71K/63.8K)                                      |

**結構性洞察**：

1. Threads bias 是**默認**、不是**絕對**。
2. 反轉條件：(a) 小眾 / 聽覺 / 意境型 類型 (b) inline 連結策略 (c) X 的 algorithmic recommendation 命中
3. 高鐵 #36 是**首次 X 對公共基建議題正面擴散 63.8K**，接近 Threads 71K，印證「**公共基建 + 具體人物（殷琪 1/5 人生）= X 破 Threads bias 的配方**」

**策略意涵**：未來類似主題應雙平台並行投放而非「Threads 為主、X 為輔」；對聽覺 / 意境型主題直接以 X inline link 為主力。

### 🔥 Finding 2：孢子曲線呈現**「尖峰高原慢衰減」三段式**（既非冪次、也非 S 曲線）

| 孢子               | 2h           | 6h        | D+1           | D+2       | D+4        | D+7+                              | 形狀               |
| ------------------ | ------------ | --------- | ------------- | --------- | ---------- | --------------------------------- | ------------------ |
| #29 李洋 Threads   | 21K          | 130K      | 180K (D+0.33) | —         | 300K       | **300K (D+7)**                    | 尖峰 → 高原 → 飽和 |
| #30 李洋 X         | —            | —         | 112K (~29h)   | —         | 135K       | **136K (D+7)**                    | 尖峰小 → 飽和高    |
| #25 張懸 Threads   | —            | —         | —             | —         | 120K (D+4) | **190K (D+8)**                    | 慢熱 → 長尾擴散    |
| #22 鄭麗文 Threads | —            | —         | —             | —         | —          | **49K (D+10)**                    | 早期飽和           |
| #33 草東 Threads   | 10K (D+0.25) | 16K (D+1) | 20K (D+2)     | 28K (D+3) | —          | 穩步擴散（無尖峰）                |
| #36 高鐵 X         | 3            | —         | 48K (D+1)     | 64K (D+2) | —          | **爆發型**（edit 後 v2 URL 接力） |

**三段式模型**：

1. **尖峰期（0-6h）**：取決於初始觸達。李洋 6h 130K 定調 blockbuster，Cicada 6h ~100 views 定調 niche
2. **高原期（6h-D+4）**：持續 +20-25K/h 的穩定擴散（李洋 #29 8h→D+4 從 180K → 300K）
3. **飽和期（D+4-D+7+）**：李洋 #29 Threads 到 D+7 仍是 300K，沒有進一步成長——**真正飽和**

**破局案例**：#30 李洋 X **從 D+4 135K 到 D+7 136K** 只增 0.7%，**比 Threads 更早飽和**——X 算法回饋窗口更短。

**策略意涵**：D+4 檢查仍在增長代表未飽和（可等 D+7 再做最終判斷）；D+4 < 2% 成長代表已飽和可以搬進 perspectives 穩態期。

### 🔥 Finding 3：人物型 hook > 事件型 hook（222x 放大驗證）

**同日同主題雙孢子實驗（#21 vs #22）**：

- #21 鄭習會 Threads（D 時間軸型 event framing）：221 views @ D+10
- #22 鄭麗文 Threads（A 人物型 people framing）：**49,000 views** @ D+10

**221 vs 49,000 = 222x 差距**。

這印證了 Semiont EDITORIAL §3 開場原則「具體的人、具體的時刻」。事件很抽象（兩岸元首見面誰都聽過 30 年），但**人物 hook 把讀者鎖進「一個 19 歲女大學生在台大校門絕食」的 scene**，讓同一個事件產生 222x 讀者容量。

**推論**：未來時事類主題必須找到**最核心的 1 個人物視角**，把事件 framing 轉為人物 framing。「鄭麗文見習近平」的策略比「鄭習會」高 222 倍。

**解法**（已實作 γ session）：`scripts/tools/generate-dashboard-spores.py` v1.2 加「撤回/已撤回/withdrawn」欄位，+ 主表 URL 更新機制。但**X edit 後的 URL 漂移仍然依賴人工 harvest 時發現**，不是系統性保障。**建議造**：一支 `spore-url-health-check.py`，對所有 SPORE-LOG URL 做 health ping，偵測 URL 是否已 redirect / 死。

### 5.2 UTM campaign off-by-one

SPORE-LOG 有多筆備註「UTM campaign=sN 沿用（off-by-one，SPORE-LOG 實際 #N+1）」。這個 bug 長期存在，影響 GA4 campaign breakdown 的精確度。

**建議修正**：造一支 `spore-utm-audit.sh` 掃所有 SPORE-LOG 行的 UTM campaign 對應 row number，產出不一致清單。

## 六、繁殖器官的戰略體檢

### 6.1 Blockbuster ratio（hasBlockbuster 健康度）

**定義**：views > 100K 的孢子 / 總活躍孢子 = 3 / 31 = **9.7%**
（#25 張懸 190K / #29 李洋 300K / #30 李洋 X 136K）

**歷史對比**：2026-03-17 平台上線首月 blockbuster = 1/31 = 3.2%（只有 #25 安溥）。**一個月內從 3% → 9.7% = 3.1x 效率提升**。這反映了 SPORE-PIPELINE 從 v1.0 → v2.5 的 pipeline 進化價值。

### 6.2 Platform 失衡度

`Threads total / X total = 653K / 208K = 3.14x`

過去 SPORE-LOG 記載「29-510x 單條 Threads bias」只看個別孢子極端值；累積總量 **3.14x** 才是真實平台失衡度。X 的總量是 Threads 的 **32%**（並非 0.2% 那麼懸殊），這個基數還可以成長。

### 6.3 Spore cadence（發文節奏）

- 35 天發 40 孢子 = **平均 1.14 條/日**（含 #1-#3 early/empty）
- 有效 harvest 31 條跨 17 天（4/4 → 4/20）= **1.82 條/日**（發文日密度）
- **blockbuster window**：4/13-4/20（8 天內 40 → 400K+ views，佔全 revenue 50%）

**策略**：當下繁殖器官 100/100 分不應掩蓋「發文頻率仍集中在 active spike」的現實。穩態節奏應該是 1-2 條/日 長期維持，不是 spike-valley。
